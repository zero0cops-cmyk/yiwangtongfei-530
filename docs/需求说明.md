**状态色标约定（全文通用）**

| 色标 | 工单状态 | 色标 | 申报状态 |
|---|---|---|---|
| 🟠 待指派 | 待分配执行人 | 🔵 已提交 | 等待运营受理 |

## 1. 产品价值

一网统飞以「飞行全生命周期线上化」为目标，打通运营单位与委办局之间的完整服务闭环——从委办局提出飞行需求、运营单位受理调度执飞、系统识别异常生成工单，到委办局执行人员现场处置、管理员复核归档，全程线上流转、实时可查。

**委办局侧**：通过粤政易H5端，管理员可在移动端发起需求申报、派发与复核工单、监控飞行任务进度；执行人员可在现场接收工单、GPS到达确认、上传处置材料——原本需要线下沟通或PC操作的工单闭环，首次完整落地到委办局移动端。

**运营侧**：PC端新增需求池，将委办局的飞行需求申报与隐患上报统一收口，由运营人员集中受理，替代原有的多渠道沟通模式。

**渠道定位**：委办局使用H5（粤政易），运营单位使用PC；特殊场景下，运营单位可为委办局开通观察员账号，以PC只读视角支持演示与汇报。

本次变更是一网统飞服务能力向委办局侧的首次全面延伸。

## 2. 角色与渠道速查

| 角色 | 渠道 | 核心权限 | 数据范围 |
|---|---|---|---|
| 委办局管理员 | H5（粤政易） | 工单派发/复核、需求申报、隐患上报与隐患池、任务监控、数据看板、用户管理 | 本局全量 |
| 委办局执行人员 | H5（粤政易） | 工单现场执行（GPS到达确认 + 材料上传）、隐患上报 | 仅指派给自己的工单 |
| 运营人员 | PC | 全局工单管理、需求池处理（申报 + 隐患）、观察员账号创建 | 全量数据 |
| 委办局观察员 | PC | 全模块只读（汇报场景） | 本局数据·只读 |

**关键权限规则：**
- 执行人员无「任务」Tab，无需求申报权限
- 委办局管理员按局隔离，不可跨局操作
- PC端工单派发为运营人员过渡期权限，后续移交H5管理员

## 4. 主业务链路

```
【委办局管理员 · H5】需求申报
        ↓
【运营人员 · PC】需求池受理
        ↓
【运营单位 · PC】任务定义 → 飞行调度
        ↓
【系统自动】数据回传归档
        ↓
【运营单位 · PC】线索识别确认
        ↓ ← 有确认异常才继续
【运营人员PC / 管理员H5】工单派发（🟠 待指派 → 🔵 已指派）
        ↓
【委办局执行人员 · H5】现场执行
    GPS自动确认到达 → 上传现场照片 → 提交复核（🔵 已指派→跟进中→🟣 待复核）
        ↓
【委办局管理员 · H5】工单复核
    ├── 复核通过 → 🟢 已完成（终态，触发双端存档，不可逆）
    └── 打回    → 🔴 已打回 → 执行人重新处理（可无限循环）

── 独立链路（随时可触发，不依赖上方主流程）────────────────
【管理员/执行人员 · H5】隐患上报 → 【运营人员 · PC】隐患池处理（备注/删除）
```

## 5. 各角色核心操作路径

### 5.1 委办局管理员（H5）

| 场景 | 入口路径 | 关键规则 |
|---|---|---|
| 派发工单 | 工单Tab → 管理 → 工单详情 → [派发] | 仅「🟠 待指派」工单可派发；必须选本局执行人员 |
| 复核工单 | 工单Tab → 管理 → 工单详情 → [复核通过 / 打回] | 仅「🟣 待复核」工单显示操作按钮；复核通过不可逆 |
| 需求申报 | 首页 → 需求申报 → 右下角「+」新增 | 4个字段全部必填；提交后进入「🔵 已提交」状态 |
| 查看隐患 | 首页 → 隐患上报 → 列表 | 可见本局全部隐患；支持日期/类型筛选 |
| 数据看板 | 首页 | 今日/累计一键切换；飞行任务 + 飞行成果 + 工单管理三组看板；纯展示无跳转 |
| 任务监控 | 任务Tab | 执行中 / 待执行 / 已完成三子Tab；可进入航迹、实时视频、视频回放 |
| 用户管理 | 我的Tab → 用户管理 | 本局账号增删改；停用后立即无法登录；重置密码由管理员线下告知 |

### 5.2 委办局执行人员（H5）

| 场景 | 入口路径 | 关键规则 |
|---|---|---|
| 查看工单 | 工单Tab | 只看自己的工单；「待跟进」= 已指派 + 已打回合并展示 |
| GPS到达确认 | 工单详情 → 各异常点 | **进入围栏自动确认，无需手动点击**；GPS未授权则无法确认 |
| 上传现场材料 | GPS确认后自动展开 | 至少1张照片必填；文字说明选填（最多500字）；内容本地自动保存 |
| 提交复核 | 工单详情底部 → [提交复核] | **所有异常点均已GPS确认 + 各自至少1张照片**，按钮才可点击；否则置灰 |
| 隐患上报 | 首页 → 隐患上报 → 右下角「+」新增 | 类型必选（六选一）；照片必填；GPS自动获取，未授权可跳过继续提交 |

### 5.3 运营人员（PC）

| 场景 | 入口路径 | 关键规则 |
|---|---|---|
| 工单全周期管理 | 工单列表 → 7状态Tab切换 | 可对「🟠 待指派」工单发起派发；PC端无复核入口 |
| 处理需求申报 | 需求池 → 申报Tab | 受理 → 「🟡 处理中」；录入回填信息 → 「🟢 已完成」；驳回 → 「🔴 已驳回」 |
| 处理隐患 | 需求池 → 隐患Tab | 可追加备注（单次覆盖，不保留历史）；可软删除（数据保留但标记删除，不可逆） |
